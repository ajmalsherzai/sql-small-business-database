# SQL Small Business Sales & Inventory Database

## Project Overview
This project demonstrates the design and implementation of a relational SQL database for a small business sales and inventory system. The database tracks customers, products, orders, order details, payments, and inventory levels.

The goal of this project is to show practical SQL skills used in business analytics, data reporting, and database management.

## Tools Used
- SQL
- MySQL
- Relational Database Design
- Joins
- Views
- Stored Procedures
- Triggers
- Aggregate Queries

## Database Features
- Customer and order tracking
- Product and inventory management
- Payment tracking
- Sales reporting
- Revenue analysis
- Low inventory detection
- Automated inventory update using triggers

## Skills Demonstrated
- Database schema design
- Primary and foreign keys
- One-to-many relationships
- SQL joins
- Grouping and aggregation
- Views for reporting
- Stored procedures
- Triggers
- Business analytics queries

## Business Questions Answered
- Which products generate the most revenue?
- Which customers spend the most?
- What are total monthly sales?
- Which products are low in inventory?
- Which orders are unpaid or partially paid?

## Files
- `small_business_database.sql` - SQL script containing database creation, sample data, queries, view, stored procedure, and trigger.
